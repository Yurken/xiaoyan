# 小妍系统介绍（桌面端）

## 1. 系统定位

小妍是一款面向研究生与科研工作者的桌面端 AI 研究助手。

它不是单一的聊天工具，而是一套围绕科研主流程设计的研究工作台：从选题规划、文献调研、论文精读，到知识沉淀、实验记录与投稿管理，支持在同一应用内完成连续推进。

本项目采用“桌面端旗舰”策略：以本地 Tauri + Rust + SQLite 内核承载核心能力，强调隐私可控、进展可追踪、研究资产可沉淀。

从系统视角看，小妍桌面端不是”多个功能页的并列集合”，而是一个围绕科研生命周期组织的研究操作系统：

- 前台是任务流：总览 -> 规划 -> 调研 -> 阅读 -> 沉淀 -> 实验 -> 投稿。
- 中台是能力流：检索、分析、写作、协同、版本管理、质量诊断。
- 后台是资产流：论文、知识、会话、checkpoint、实验、投稿记录统一沉淀并可复用。

## 2. 桌面端产品目标

小妍桌面端重点解决三类科研痛点：

- 研究流程割裂：规划、检索、阅读、写作、投稿分散在多个工具中，切换成本高。
- 研究资产难沉淀：论文、笔记、对话、实验结论难以统一管理与复用。
- AI 使用不可解释：仅有结果，没有过程与来源，难以建立信任。

对应地，桌面端提供三个核心价值：

- 一体化工作流：用统一界面串联科研全链路。
- 结构化沉淀：把论文、知识、会话、实验、投稿记录沉淀为可检索资产。
- 可观测 AI 协作：支持查看规划步骤、执行轨迹与中间产物，提升可解释性。

进一步细化为五个可落地目标：

- 连续推进：避免“每到下一步就换工具、丢上下文”。
- 研究资产化：把临时讨论、阅读结论、实验结果转为长期资产。
- 决策可解释：重要 AI 结果应可追溯来源、步骤与上下文。
- 协作可重放：会话、计划、版本与审稿意见可回顾、可复盘。
- 架构可演进：新能力可插入既有流程，而不破坏已有模块边界。

## 3. 桌面端功能全景

桌面端导航包含以下模块：工作台、规划、综述、论文、知识、小妍、实验、投稿、工具、设置。

### 3.1 工作台（首页）

工作台是研究进展总览入口，也是默认进入的第一个页面，聚合展示：

- 研究主题总览：按研究方向聚合路线、论文、笔记、会话、checkpoint、知识主张、实验与关联投稿，提供统一视图识别当前进展
- 继续研究入口：按最近活动、checkpoint 和下一步建议排序，帮助用户从首页快速接回当前研究
- 论文库规模与分析进度
- 研究方向数量与规划状态
- 知识笔记沉淀情况
- 对话会话沉淀情况
- 投稿相关统计（进行中任务、待回复审稿意见）

并提供快速操作入口，支持从首页直接进入规划、综述、论文导入、小妍协作与工具箱。

### 3.2 研究方向规划（规划）

用户输入研究主题后，系统可辅助梳理：

- 学习路线与阶段拆分
- 关键文献与代表问题
- 可行切入点与后续行动建议

该模块适合用于早期选题探索与中期方向收敛。

### 3.3 文献综述（综述）

围绕用户问题进行文献调研，支持：

- 文献检索与摘要阅读
- 结构化综述初稿生成
- 与研究方向目录联动沉淀

该模块用于快速形成可讨论的综述骨架，缩短从“找资料”到“形成文本”的时间。

### 3.4 论文库与精读分析（论文）

论文模块是桌面端的核心资产中心，支持：

- PDF 导入与管理
- 按研究方向分组管理论文
- 论文标签、排序与筛选
- 论文精读分析（文字 + 图表并排呈现，AI 积极引用图表编号）
- 图表/表格引用关联展示
- 论文信息识别与编辑
- 解析质量链路：记录每次解析的运行信息（解析器、耗时、正文长度、章节/图表估算、回退路径与错误信息），支持重解析并保留历史解析记录

这一模块将”文件管理 + AI 精读 + 可追溯阅读记录 + 解析质量审计”整合在同一空间。

### 3.5 知识库（知识图谱 + 知识笔记）

知识模块提供两种互补视图：

- 知识图谱视图：组织观点、证据与关联关系
- 笔记视图：沉淀结构化知识条目并支持关联管理

支持按研究方向聚焦，帮助用户回答“结论来自哪里、与哪些证据相关”。

### 3.6 小妍协同台（小妍）

小妍协同台是多 Agent 协作入口，基于 `AgentRuntime` 实现统一的工具注册、上下文管理与事件边界。现有协同流程以 `XiaoyanNative` 为默认路径运行，普通用户路径只呈现”小妍”作为唯一助手身份，不暴露内部 Agent 标识。

支持：

- 会话管理与研究上下文绑定（聚合路线、论文、笔记、知识图谱主张与最近 checkpoint）
- 流式对话与结果生成
- 任务计划步骤展示
- Agent 运行轨迹与状态可视化
- 技能（Skill）选择与协作
- 长期记忆 checkpoint：会话完成或失败后自动记录轻量 checkpoint，捕获关键决策、开放问题与下一步建议，并在工作台续接入口和主题总览中展示

该模块强调”过程可见”，不仅给出答案，也展示推进过程，并对关键节点留痕以便后续续接。

### 3.7 实验工作区（实验）

实验模块用于沉淀复现与验证过程，支持：

- 实验记录创建与编辑
- 配置、结果与备注管理
- 图片/截图附件上传与预览
- 与投稿条目关联追踪

该模块把零散实验日志转为可追溯资产，便于复盘与写作取证。

### 3.8 投稿管理（投稿）

投稿模块覆盖投稿生命周期管理，包含：

- 刊会追踪（会议/期刊）
- 投稿看板（Kanban）
- 提交前检查清单
- 版本管理与对比
- AI 多视角预审：按方法、实验、写作等职责生成分类标签与结论（大修/小修/接收/拒稿），结果保存为投稿前诊断报告
- 诊断证据链：诊断报告可转为 checklist 或修改任务，并关联论文版本与实验记录，形成”审稿 → 诊断 → 修改 → 验证”闭环
- 审稿轮次与意见归档
- Cover Letter 与文本润色辅助
- 投稿时间线与拒稿转投恢复

该模块将”研究产出到投稿执行”纳入同一流程闭环，并通过诊断证据链关联实验与版本，让投稿前自检和修改计划更连贯。

### 3.9 实用工具（工具）

工具模块聚合高频科研工具能力：

- 论文检索（含 arXiv 检索）
- 刊会/分区信息查询
- 学术翻译
- Markdown 整理
- PPT 生成与导出
- 科研友链导航

用于减少在外部网站与零散工具间的切换。

### 3.10 系统设置（设置）

设置模块提供系统级配置与运行管理，支持：

- 模型与连接配置（多 Provider）
- 角色分工与任务模型策略
- 主题与布局偏好
- 记忆与技能管理
- 配置加密导入/导出
- 更新检查与版本信息

让用户能按科研任务类型定制小妍能力分工。

## 4. 系统分层与职责边界

桌面端建议按四层理解，便于团队开发与后续扩展：

### 4.1 交互编排层（UI 页面层）

- 代表模块：工作台、规划、综述、论文、知识、小妍、实验、投稿、工具、设置。
- 核心职责：承接用户意图、组合功能区块、呈现结果与状态。
- 边界约束：页面负责“编排”，不承载大段跨域业务逻辑。

### 4.2 领域能力层（Feature 层）

- 代表能力：论文管理、知识图谱、投稿工作流、工具链、设置控制器。
- 核心职责：实现域内状态、请求编排、字段映射与可复用交互。
- 边界约束：域内共享逻辑收敛在 feature 目录，避免跨页复制。

### 4.3 AI 与后端命令层（Tauri + Rust 层）

- 代表能力：LLM 调用、RAG 检索、图谱检索、Agent 编排、数据库事务。
- 核心职责：执行计算密集或需要本地能力的任务，统一数据读写入口。
- 边界约束：命令层负责能力提供，不直接耦合页面表现。

### 4.4 资产持久层（SQLite 层）

- 核心职责：保存研究对象、过程对象、配置对象与关系对象。
- 边界约束：所有核心科研资产可落库、可追踪、可回放。

## 5. 模块输入输出与上下游关系

下面按“输入 -> 处理 -> 输出 -> 下游”描述桌面端核心模块。

### 5.1 工作台

- 输入：研究方向、论文分析、知识笔记、会话、checkpoint、投稿与实验统计、知识主张。
- 处理：多源聚合、主题总览组装、进展指标计算、续接入口排序。
- 输出：研究态势总览、主题进展视图、下一步建议与行动入口。
- 下游：规划、综述、论文、小妍、投稿、工具。

### 5.2 规划

- 输入：研究主题、关键词、用户背景与目标。
- 处理：方向拆解、学习路径生成、问题域聚焦。
- 输出：研究方向对象、路线建议、候选文献线索。
- 下游：综述、论文、知识、小妍。

### 5.3 综述

- 输入：研究问题、方向上下文、检索结果。
- 处理：摘要阅读、证据聚合、结构化写作。
- 输出：综述草稿、候选论文列表、关键争议点。
- 下游：论文、知识、小妍、投稿（写作素材）。

### 5.4 论文

- 输入：PDF 文件、研究方向归属、用户标签。
- 处理：解析、分块、分析、图表关联、元信息编辑。
- 输出：论文元数据、分析文本、图表资产、可追溯阅读记录。
- 下游：知识、小妍、实验、投稿。

### 5.5 知识

- 输入：论文分析结论、笔记条目、证据链接、方向上下文。
- 处理：知识节点组织、关系构建、图谱过滤与追溯。
- 输出：知识图谱视图、结构化笔记与证据链。
- 下游：小妍、综述、投稿（论证材料）。

### 5.6 小妍

- 输入：会话上下文、方向上下文、知识与论文引用、checkpoint 历史、技能选择。
- 处理：AgentRuntime 调度与执行、流式生成、状态可视化、checkpoint 记录。
- 输出：可解释回答、计划步骤、Agent 运行记录、中间产物、checkpoint（关键决策、开放问题、下一步建议）。
- 下游：规划、综述、论文、知识、投稿、实验（反向驱动各模块）；工作台（通过 checkpoint 提供续接入口）。

### 5.7 实验

- 输入：实验配置、结果描述、附件、关联投稿条目。
- 处理：实验记录版本化、结果归档、图像证据挂载。
- 输出：实验日志、证据附件、与论文/投稿关联索引。
- 下游：论文、知识、投稿。

### 5.8 投稿

- 输入：论文版本、审稿意见、目标刊会、实验与论证材料。
- 处理：DDL 跟踪、看板推进、版本管理、审稿归档、AI 多视角预审。
- 输出：投稿状态、版本历史、审稿轮次、回复素材。
- 下游：工作台（进度反馈）、小妍（回复与改稿协同）。

### 5.9 工具

- 输入：检索条件、翻译文本、Markdown 文稿、PPT 主题。
- 处理：任务型工具执行与结果生成。
- 输出：检索候选、翻译结果、格式化文本、演示文稿。
- 下游：综述、论文、投稿、知识（作为辅助输入）。

### 5.10 设置

- 输入：模型连接参数、任务分工策略、视觉主题偏好。
- 处理：配置校验、连接测试、策略持久化、导入导出。
- 输出：系统运行参数、能力路由策略。
- 下游：全模块（作为统一运行基线）。

## 6. 模块关系矩阵（核心依赖）

说明：A -> B 表示 A 的结果会显著影响 B 的输入或行为。

- 工作台 -> 规划、综述、论文、小妍、投稿、工具：提供导航与状态驱动。
- 规划 -> 综述、论文、知识、小妍：提供主题边界与语义上下文。
- 综述 -> 论文、知识、小妍、投稿：提供候选文献与写作骨架。
- 论文 -> 知识、小妍、实验、投稿：提供核心证据与分析资产。
- 知识 -> 小妍、综述、投稿：提供结构化论证与可追溯关系。
- 小妍 -> 全模块：作为跨域协同与任务编排入口，反向驱动模块动作。
- 实验 -> 论文、知识、投稿：提供复现结果与实证证据。
- 投稿 -> 工作台、小妍：反馈任务状态并触发改稿协同。
- 工具 -> 综述、论文、投稿、知识：提供任务型外部能力补充。
- 设置 -> 全模块：控制模型、路由、主题与运行策略。

## 7. 关键跨模块流程

### 7.1 选题到综述流程

1. 用户在规划中定义研究方向与关键词。
2. 规划结果作为综述的检索语义输入。
3. 综述产出候选论文与结构化初稿。
4. 候选论文进入论文库形成可分析对象。

价值：把“方向想法”快速转化为“可阅读、可讨论的文献集合”。

### 7.2 阅读到知识沉淀流程

1. 在论文模块导入 PDF 并执行分析。
2. 分析结论与关键图表进入知识模块。
3. 用户补充笔记并建立证据关系。
4. 图谱结果反向用于后续问答与写作。

价值：把“单篇阅读结果”转化为“跨论文可复用知识网络”。

### 7.3 知识到投稿执行流程

1. 投稿模块选择目标刊会并建立版本节奏。
2. 从知识模块提取论证与证据链用于稿件增强。
3. 审稿意见进入归档后，调用小妍生成回复草案。
4. 新版本与回复策略再次沉淀为可追踪资产。

价值：把“写作与审稿”从临时应对升级为可复盘过程。

### 7.4 小妍驱动的跨模块协同流程

1. 用户在小妍中提出复合任务（如“补综述并准备投稿答复”）。
2. 小妍基于会话上下文选择相关能力模块。
3. 过程展示计划步骤、执行状态与中间产物。
4. 结果回写到对应模块（综述、知识、投稿等）。

价值：把多步骤研究任务从“手工切换执行”转为“可观测协同执行”。

## 8. 数据对象与归属关系

桌面端建议把数据对象划分为四类，明确所有权与生命周期。

### 8.1 研究对象（长期资产）

- 研究方向
- 论文与论文分析（含解析质量记录）
- 知识笔记与图谱关系

特征：生命周期长，跨任务复用频率高。

### 8.2 过程对象（任务轨迹）

- 会话消息
- Agent 运行记录与 checkpoint
- 计划步骤与中间产物

特征：承载可解释性与复盘能力，checkpoint 同时驱动续接入口和主题总览。

### 8.3 交付对象（产出导向）

- 实验记录与附件
- 投稿条目、版本、审稿意见
- 投稿前诊断报告（含修改任务与 checklist 关联）
- Cover Letter 与回复文本

特征：直接服务于论文交付与投稿执行，诊断报告充当"审稿反馈—修改任务—实验验证"的证据桥梁。

### 8.4 系统对象（运行基线）

- 模型连接参数
- 路由与角色分工配置
- 主题与布局偏好

特征：影响全局行为，不直接构成研究结论。

## 9. 系统状态流转（运行视角）

桌面端主要存在三条状态流：

### 9.1 任务状态流

待开始 -> 执行中 -> 已完成/失败 -> 复盘沉淀

典型承载：小妍计划、投稿看板、工具任务。

### 9.2 内容状态流

草稿 -> 结构化 -> 版本化 -> 归档

典型承载：综述文本、论文分析、投稿版本、审稿回复。

### 9.3 证据状态流

原始材料 -> 解析提取 -> 关系链接 -> 引用复用

典型承载：论文图表、知识证据链、实验附件。

## 10. 核心技术架构（桌面端）

桌面端采用前后端同机架构：

- 前端：React + TypeScript + Vite
- 桌面容器：Tauri v2
- 后端能力层：Rust（AgentRuntime 编排 + 业务 Command + 服务层抽象）
- 数据层：本地 SQLite

典型调用链路为：React UI -> Tauri invoke/listen -> Rust 业务命令 -> SQLite 持久化。多 Agent 协同链路为：用户输入 -> AgentRuntime 调度 -> AgentGraph 状态机 -> 各 Agent 节点执行 -> 流式返回结果并记录 checkpoint。

该架构具备以下优势：

- 本地运行：关键数据保存在本机，减少外部依赖。
- 响应稳定：核心流程不依赖浏览器多页切换。
- 可扩展：便于持续增加新功能域（规划、投稿、实验、工具等）。

## 11. 模块协同中的技术要点

### 11.1 统一调用通道

- 前端通过统一 client 发起请求。
- 通过 Tauri invoke/listen 与 Rust 命令层交互。
- 事件监听用于承接流式或阶段性结果。

收益：跨模块协同时请求语义一致，便于追踪与排障。

### 11.2 本地优先的数据闭环

- 核心资产本地落库，支持离线可读与持续沉淀。
- 模块之间通过共享数据对象建立关联，而非页面间直接耦合。

收益：模块解耦更强，资产复用路径更清晰。

### 11.3 可解释协同机制

- 小妍侧保留计划步骤、执行状态与中间产物。
- 投稿与实验侧保留版本与证据材料。

收益：结果不是“黑箱瞬时输出”，而是“可追溯研究过程”。

## 12. 数据与安全

桌面端遵循“研究资产本地沉淀”原则：

- 论文、笔记、会话、实验与投稿数据主要存储在本地 SQLite。
- 设置支持加密导入/导出，便于设备迁移与备份。
- 敏感字段（如 API Key）在读取展示时采用掩码策略。

这使系统在易用性和隐私可控之间保持平衡。

## 13. 典型使用流程（桌面端）

一个常见科研闭环可按以下路径进行：

1. 在”工作台”中查看研究主题总览，通过 checkpoint 续接之前的研究进度。
2. 在”规划”中确定研究方向与问题边界。
3. 在”综述”中生成文献调研草稿并筛选候选论文。
4. 在”论文”中导入 PDF，完成精读分析（含图表抽取），解析质量链路记录每次解析详情。
5. 在”知识”中沉淀观点与证据关系，形成可复用知识图谱。
6. 在”小妍”中通过 AgentRuntime 调度多 Agent 协同问答与任务推进，会话完成时自动生成 checkpoint。
7. 在”实验”中记录复现实验配置、结果与截图，与投稿条目关联。
8. 在”投稿”中推进版本管理，运行 AI 多视角预审生成诊断报告并转为修改任务，关联论文版本与实验记录形成证据链闭环。

该流程的核心价值是”连续推进 + 全程留痕 + 关键节点 checkpoint + 可回溯复用”。

## 14. 总结

小妍桌面端并非简单功能集合，而是围绕科研全过程构建的系统化工作台。

它通过本地化架构、模块化能力和可观测 AI 协作机制，帮助用户把“临时操作”转化为“长期资产”，把“碎片任务”转化为“可持续推进的研究流程”。

当系统进入多人协作或更复杂研究场景时，这种“模块可分工、数据可贯通、过程可解释”的设计会持续放大价值：

- 对用户：减少重复劳动，提升研究节奏连续性。
- 对团队：降低功能扩展时的耦合风险。
- 对系统：形成可维护、可演进、可复盘的科研操作底座。
