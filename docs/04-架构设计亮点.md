# 小妍 — 架构设计亮点

## 一、五层分层架构

小妍桌面端按五层理解，模块边界清晰：

```
┌─────────────────────────────────────────────┐
│  交互编排层（UI 页面层）                       │
│  工作台 / 规划 / 综述 / 论文 / 知识 / 小妍     │
│  实验 / 投稿 / 工具 / 设置 / 应用锁            │
├─────────────────────────────────────────────┤
│  领域能力层（Feature 层）                      │
│  论文管理 / 知识图谱 / 投稿工作流 / 工具链      │
│  桌面伴侣 / 应用锁定 / 记忆隐私                │
├─────────────────────────────────────────────┤
│  服务层（Services）                           │
│  对话上下文 / 记忆隐私 / 记忆检索 / 设置管理    │
│  来源管理 / 投稿管理                          │
├─────────────────────────────────────────────┤
│  AI 与后端命令层（Tauri + Rust）               │
│  LLM / RAG / Graph RAG / Agent 编排 / 数据库   │
├─────────────────────────────────────────────┤
│  资产持久层（SQLite）                          │
│  研究对象 / 过程对象 / 交付对象 / 系统对象      │
└─────────────────────────────────────────────┘
```

**边界约束**：
- 页面只负责"编排"，不承载跨域业务逻辑
- Feature 层域内共享，禁止跨页复制
- 服务层封装业务逻辑，命令层只做参数校验与调度
- 命令层提供能力，不耦合页面表现
- 所有核心资产可落库、可追踪、可回放

## 二、前后端同机架构

### 2.1 调用链路

```
React UI → Tauri invoke/listen → Rust 业务命令 → SQLite 持久化
```

### 2.2 核心后端能力模块

Rust 后端按能力域组织，核心模块包括：

| 能力域 | 职责 | 代码量级 |
|---|---|---|
| LLM 客户端（含传输抽象与视觉支持） | 多服务商统一接入、SSE 流式管道、视觉模型调用 | ~38KB |
| RAG 向量检索 | 文本分块 + 余弦相似度向量检索 | ~8KB |
| Graph RAG 图增强检索 | 检索时融合图邻域上下文，支持多跳推理 | ~12KB |
| 引用关系图引擎 | 论文引用网络构建（节点 + 有向边） | ~13KB |
| Agent 状态图编排 | Supervisor 显式状态机，节点调度与路由 | ~21KB |
| Agent 节点实现 | 各专业 Agent（检索、规划、综述、复现等） | ~13KB |
| 数据库与 Schema 迁移 | SQLite 初始化、表结构演进 | ~25KB |
| 应用状态管理 | 线程安全的全局状态（Arc + RwLock） | ~14KB |
| CCF 目录索引 | 内嵌 679 条会议/期刊索引与查询 | ~8KB |
| 期刊分区索引 | 内嵌 22804 条期刊数据与动态过滤 | ~10KB |
| 联网搜索 | 外部搜索结果获取与注入 | — |
| 提示词管理 | Agent 系统提示词与角色定义 | — |

### 2.3 后端能力域总览

后端按功能划分为 27 个能力模块，覆盖以下领域：

| 领域 | 覆盖能力 |
|---|---|
| **设置与系统** | 设置读写（事务）、加密导入/导出、连接测试、模型列表、配置历史、自动更新 |
| **论文处理** | PDF 上传/解析、文本提取、双路图表提取、产物管理、语义搜索、AI 重命名 |
| **AI 分析** | 论文精读（多维分析）、复现指南生成、AI 模拟审稿 |
| **知识管理** | 研究方向 CRUD、知识笔记 CRUD、知识图谱构建与查询、语义搜索 |
| **记忆体系** | 手动/自动记忆录入与检索、上下文构建、记忆隐私密码保护与分层访问 |
| **Agent 协同** | 多 Agent 编排与流式对话、Agent 执行记录与产物管理 |
| **检索工具** | arXiv 三步级联检索 + LLM 重排、期刊分区查询、CCF 目录查询、联网搜索 |
| **投稿工作流** | 投稿 CRUD、版本管理、审稿归档、Cover Letter、拒稿转投 |
| **实验管理** | 实验记录 CRUD、附件管理、投稿关联 |
| **辅助能力** | 学术翻译、Markdown 格式化、PPT 生成、Obsidian 导出、技能管理 |
| **安全与隐私** | 应用锁（密码 + 超时）、记忆隐私分层、配置加密 |

## 三、数据对象四层分类

### 3.1 研究对象（长期资产）

- 研究方向、论文与论文分析、知识笔记与图谱关系
- 特征：生命周期长，跨任务复用频率高

### 3.2 过程对象（任务轨迹）

- 会话消息、Agent 运行记录、计划步骤与中间产物
- 特征：承载可解释性与复盘能力

### 3.3 交付对象（产出导向）

- 实验记录与附件、投稿条目/版本/审稿意见、Cover Letter
- 特征：直接服务于论文交付与投稿执行

### 3.4 系统对象（运行基线）

- 模型连接参数、路由与角色分工配置、主题与布局偏好
- 特征：影响全局行为，不直接构成研究结论

## 四、三条核心状态流

### 4.1 任务状态流

```
待开始 → 执行中 → 已完成/失败 → 复盘沉淀
```

承载：小妍计划、投稿看板、工具任务

### 4.2 内容状态流

```
草稿 → 结构化 → 版本化 → 归档
```

承载：综述文本、论文分析、投稿版本、审稿回复

### 4.3 证据状态流

```
原始材料 → 解析提取 → 关系链接 → 引用复用
```

承载：论文图表、知识证据链、实验附件

## 五、模块关系矩阵

```
工作台 ──→ 规划、综述、论文、小妍、投稿、工具（导航与状态驱动）
规划 ────→ 综述、论文、知识、小妍（提供主题边界与语义上下文）
综述 ────→ 论文、知识、小妍、投稿（提供候选文献与写作骨架）
论文 ────→ 知识、小妍、实验、投稿（提供核心证据与分析资产）
知识 ────→ 小妍、综述、投稿（提供结构化论证与可追溯关系）
小妍 ────→ 全模块（跨域协同入口，反向驱动各模块）
实验 ────→ 论文、知识、投稿（提供复现结果与实证证据）
投稿 ────→ 工作台、小妍、实验（反馈任务状态并触发改稿协同）
工具 ────→ 论文、知识、投稿（检索、翻译、PPT、导出等能力输出）
```

## 六、跨端架构策略

```
┌──────────────────────────────────────────────┐
│  Desktop（旗舰端）                             │
│  Tauri / Rust / SQLite — 全功能，本地内核       │
├──────────────────────────────────────────────┤
│  Web（展示/协作版）                             │
│  HTTP 后端 — 功能子集，远程协作                 │
├──────────────────────────────────────────────┤
│  Mobile（轻量陪伴端）                           │
│  HTTP 后端 — 消费型场景（阅读、通知）           │
└──────────────────────────────────────────────┘
```

**原则**：新能力默认落在 Desktop，Web/Mobile 按需选择性同步，不追求三端对齐。

## 七、数据持久化设计

### 7.1 数据持久化体系

SQLite 数据库按数据资产类型组织：

| 数据类别 | 持久化内容 |
|---|---|
| **系统配置** | 应用设置键值对（含模型参数、路由规则、UI 偏好） |
| **论文资产** | 论文元数据 + 全文 + 分块向量 + 图表文件 + AI 分析结果 + 复现指南 |
| **研究方向** | 研究主题 + 学习路线 + 阶段拆解 |
| **知识资产** | 知识笔记 + Embedding 向量 + 图谱节点与关系 |
| **用户记忆** | 手动备忘 + 自动操作记录 + AI 过程观察 |
| **对话记录** | 会话元数据 + 消息历史 |
| **Agent 轨迹** | 执行记录 + 中间产物 + 计划步骤 |
| **投稿资产** | 投稿条目 + 版本快照 + 审稿意见 + Cover Letter |
| **实验资产** | 实验记录 + 附件 + 投稿关联 |

### 7.2 设置持久化机制

- **写入**：事务内 upsert，成功后同步更新内存缓存
- **读取**：启动时全量加载，合并进内存缓存（默认值兜底）
- **敏感字段**：读取返回 `***` 占位，保存跳过 `***` 值
- **持久性**：卸载/重装 App 时数据库保留
- **迁移**：支持加密导出 `.rcconf`（AES-256-GCM + PBKDF2）
