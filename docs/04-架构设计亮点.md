# 小妍 — 架构设计亮点

## 一、四层分层架构

小妍桌面端按四层理解，模块边界清晰：

```
┌─────────────────────────────────────────────┐
│  交互编排层（UI 页面层）                       │
│  工作台 / 规划 / 综述 / 论文 / 知识 / 小妍     │
│  实验 / 投稿 / 工具 / 设置                     │
├─────────────────────────────────────────────┤
│  领域能力层（Feature 层）                      │
│  论文管理 / 知识图谱 / 投稿工作流 / 工具链      │
├─────────────────────────────────────────────┤
│  AI 与后端命令层（Tauri + Rust）               │
│  LLM / RAG / Graph RAG / Agent 编排 / 数据库   │
├─────────────────────────────────────────────┤
│  资产持久层（SQLite）                          │
│  研究对象 / 过程对象 / 交付对象 / 系统对象      │
└─────────────────────────────────────────────┘
```

**边界约束**：
- 页面只负责"编排"，不承载跨域业务逻辑
- Feature 层域内共享，禁止跨页复制
- 命令层提供能力，不耦合页面表现
- 所有核心资产可落库、可追踪、可回放

## 二、前后端同机架构

### 2.1 调用链路

```
React UI → Tauri invoke/listen → Rust 业务命令 → SQLite 持久化
```

### 2.2 核心 Rust 模块

| 模块 | 职责 | 规模 |
|---|---|---|
| `llm.rs` | LLM 客户端（OpenAI / Anthropic / 兼容接口，SSE 流式；含视觉模型） | ~38K |
| `rag.rs` | 文本分块 + 余弦相似度向量检索 | ~8K |
| `citation_graph.rs` | 论文引用关系图（节点 + 有向边） | ~13K |
| `graph_rag.rs` | Graph RAG：检索时融合图邻域上下文 | ~12K |
| `agent_graph.rs` | 小妍 Agent 状态图（显式状态机） | ~21K |
| `agent_nodes.rs` | 各 Agent 节点实现 | ~13K |
| `db.rs` | SQLite 初始化与 Schema 迁移 | ~25K |
| `state.rs` | AppState（线程安全 Arc<RwLock>） | ~14K |
| `ccf.rs` | CCF 目录索引与查询（679 条） | ~8K |
| `journal_partitions.rs` | 期刊分区索引与查询（22804 条） | ~10K |

### 2.3 命令模块

```
commands/
├── settings        设置读写（事务）、加密导入/导出
├── papers          PDF 上传、解析、图表提取、分析、复现、AI 重命名
├── knowledge       研究方向 + 知识笔记
├── knowledge_graph 知识图谱构建与查询
├── memory          记忆管理（手动 / 自动提取 / 上下文构建）
├── chat            多 Agent 编排与流式对话
├── arxiv           arXiv 检索与 LLM 重排
├── journal         期刊分区查询 + 按等级/学科过滤
├── submission      投稿管理、版本控制、审稿归档
└── misc            规划器、综述生成、搜索
```

## 三、数据对象四层分类

### 3.1 研究对象（长期资产）

- 研究方向、论文与论文分析、知识笔记与图谱关系
- 特征：生命周期长，跨任务复用频率高

### 3.2 过程对象（任务轨迹）

- 会话消息、Agent 运行记录、计划步骤与中间产物
- 特征：承载可解释性与复盘能力

### 3.3 交付对象（产出导向）

- 实验记录与附件、投稿条目/版本/审稿意见、Cover Letter
- 特征：直接服务于论文交付与投稿执行

### 3.4 系统对象（运行基线）

- 模型连接参数、路由与角色分工配置、主题与布局偏好
- 特征：影响全局行为，不直接构成研究结论

## 四、三条核心状态流

### 4.1 任务状态流

```
待开始 → 执行中 → 已完成/失败 → 复盘沉淀
```

承载：小妍计划、投稿看板、工具任务

### 4.2 内容状态流

```
草稿 → 结构化 → 版本化 → 归档
```

承载：综述文本、论文分析、投稿版本、审稿回复

### 4.3 证据状态流

```
原始材料 → 解析提取 → 关系链接 → 引用复用
```

承载：论文图表、知识证据链、实验附件

## 五、模块关系矩阵

```
工作台 ──→ 规划、综述、论文、小妍、投稿、工具（导航与状态驱动）
规划 ────→ 综述、论文、知识、小妍（提供主题边界与语义上下文）
综述 ────→ 论文、知识、小妍、投稿（提供候选文献与写作骨架）
论文 ────→ 知识、小妍、实验、投稿（提供核心证据与分析资产）
知识 ────→ 小妍、综述、投稿（提供结构化论证与可追溯关系）
小妍 ────→ 全模块（跨域协同入口，反向驱动各模块）
实验 ────→ 论文、知识、投稿（提供复现结果与实证证据）
投稿 ────→ 工作台、小妍（反馈任务状态并触发改稿协同）
```

## 六、跨端架构策略

```
┌──────────────────────────────────────────────┐
│  Desktop（旗舰端）                             │
│  Tauri / Rust / SQLite — 全功能，本地内核       │
├──────────────────────────────────────────────┤
│  Web（展示/协作版）                             │
│  HTTP 后端 — 功能子集，远程协作                 │
├──────────────────────────────────────────────┤
│  Mobile（轻量陪伴端）                           │
│  HTTP 后端 — 消费型场景（阅读、通知）           │
└──────────────────────────────────────────────┘
```

**原则**：新能力默认落在 Desktop，Web/Mobile 按需选择性同步，不追求三端对齐。

## 七、数据持久化设计

### 7.1 数据库表结构

| 表 | 说明 |
|---|---|
| `settings` | 应用设置键值对 |
| `papers` | 论文元数据 + 全文 |
| `paper_chunks` | 论文分块 + Embedding 向量（JSON 存储） |
| `paper_figures` | 论文图表文件 |
| `paper_analyses` | 论文 AI 分析结果 |
| `reproduction_guides` | 论文复现指南 |
| `research_interests` | 研究方向 + 学习路线 |
| `knowledge_notes` | 知识笔记 + Embedding |
| `memories` | 用户记忆（手动 / 自动） |
| `chat_sessions` | 对话会话 |
| `chat_messages` | 对话消息 |
| `agent_runs` | Agent 执行记录 |
| `agent_artifacts` | Agent 产物 |

### 7.2 设置持久化机制

- **写入**：事务内 upsert，成功后同步更新内存缓存
- **读取**：启动时全量加载，合并进内存缓存（默认值兜底）
- **敏感字段**：读取返回 `***` 占位，保存跳过 `***` 值
- **持久性**：卸载/重装 App 时数据库保留
- **迁移**：支持加密导出 `.rcconf`（AES-256-GCM + PBKDF2）
