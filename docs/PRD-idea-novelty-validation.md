# 规划：小妍 idea 创新性验证增强

- 状态：提案，待评审
- 版本：0.5.x（建议作为后续迭代）
- 负责人：产品 / Desktop / AI（待指定）
- 相关模块：选题规划、文献综述、论文库、知识图谱、Copilot 多 Agent

## 1. 摘要

当前小妍的「想 idea」功能（让小妍推荐课题、给小妍资料提炼 idea）主要依赖 LLM 的 prompt 约束来生成「看起来具体、前沿」的研究方向，但**未与已有文献做比对**。这导致生成的 idea 可能已经被发表过，用户若直接据此开题或投稿，存在撞题风险。

本规划目标是在现有 idea 生成流程中增加一层**创新性校验**：对生成的 idea，自动在本地论文库和外部学术源中检索相似工作，并给出「已有相似研究」与「潜在差异化切入点」的提示，让用户自己判断创新性。

## 2. 背景与问题

### 2.1 现状

现有两个 idea 生成入口：

1. **让小妍帮你找方向**（`knowledge_suggest_topics`）
   - 输入：研究领域、目标类型、个人背景
   - 输出：4~5 个课题方向 + 推荐理由
   - 创新依据：prompt 要求「体现近两年学术前沿」「能发顶会/顶刊」

2. **给小妍资料找 idea**（`knowledge_ideas_from_materials`）
   - 输入：文字、txt/md/pdf 文本、图片
   - 输出：4~6 个 idea（title / rationale / background / keywords）
   - 创新依据：prompt 要求「体现近两年学术前沿或落地价值」

### 2.2 问题

LLM 生成文本时只能依赖训练数据中的模式，**无法实时、准确地判断**：

- 某个具体方法是否已被某篇论文发表
- 某个 claim 是否已经被证过
- 某个组合是否只是简单拼接而无新意

因此当前功能更适合做「灵感发散」和「选题探索」，不能作为创新性终审。

## 3. 目标

### 3.1 产品目标

在 idea 生成后，为用户提供一轮**轻量级的创新性预检**：

1. 自动用 idea 的 keywords / title 在本地论文库中检索最相关的已有工作；
2. 可选地在外部学术源（arXiv / Semantic Scholar）中检索相似研究；
3. 让 LLM 对比「生成的 idea」与「检索到的已有工作」，输出：
   - 相似点（可能撞题的风险）
   - 差异点（可继续深挖的方向）
   - 建议下一步（补做文献、换数据集、换方法角度等）
4. 把结果以卡片形式嵌入 idea 详情，不阻塞用户，由用户自主决定是否继续。

### 3.2 非目标

- 不做自动化的「创新性分数」或「能否中顶会」的断言；
- 不承诺覆盖所有学术数据库；
- 不替代用户和导师的专业判断；
- 不在首次生成 idea 时强制等待外部检索完成（避免阻塞）。

### 3.3 关键结果

| KR | 目标 |
| --- | --- |
| 覆盖率 | 生成的 idea 中，至少 80% 能成功触发本地相似性检查 |
| 相关性 | 检索返回的 Top-3 结果中，至少 60% 被用户评价为「相关」 |
| 可用性 | novelty 检查卡片的平均展开率 > 40% |
| 信任 | 不因为 novelty 检查引入 P1 级隐私或 API 成本事故 |

## 4. 用户场景

| 用户任务 | 当前痛点 | 增强后的帮助 |
| --- | --- | --- |
| 研究生刚入学找方向 | LLM 给的题目听起来新，但不知道是否已有人做 | 一键看本地/外部相似工作，快速排除撞题 |
| 做开题报告前收敛 idea | 需要手动去搜每篇相关论文 | 自动生成「相似 vs 差异」对比，节省时间 |
| 基于导师讨论记录提炼 idea | 怕把导师随口说的已有方向当成新 idea | 检索后高亮已有工作，提示「该方向已有相关研究」 |

## 5. 方案设计

### 5.1 整体流程

```
用户生成 idea
    │
    ▼
[idea 卡片展示]
    │
    ▼
用户点击「检查创新性」
    │
    ├──► 本地检索：用 keywords + title 在 papers/notes 中向量+关键词检索
    │
    ├──► 外部检索（可选）：arXiv / Semantic Scholar 关键词检索
    │
    ▼
[LLM 评估 novelty]
    │
    ▼
[展示结果]
    - 相似工作列表（标题、摘要、相似原因）
    - 差异化建议
    - 风险提示
```

### 5.2 检索层

#### 本地检索

复用现有能力：

- `rag.rs`：文本分块 + 余弦相似度向量检索
- `graph_rag.rs`：知识图谱 + 引用邻域上下文融合
- SQLite 中的 `papers` 表：标题、摘要、分析结论

输入：idea 的 `title` + `keywords` + `background`
输出：Top-K 相关论文/笔记，含相似度分数

#### 外部检索

复用综述模块的 arXiv 级联检索能力：

- arXiv API：关键词搜索 + 摘要返回
- Semantic Scholar API（可选）：标题/摘要相似度

输入：idea 的 `keywords`
输出：Top-K 外部论文（标题、摘要、年份、链接）

### 5.3 评估层

新增一个 novelty 评估 prompt，输入包括：

- 生成的 idea（title / rationale / background / keywords）
- 检索到的 Top-K 相似工作（标题 + 摘要）

输出 JSON：

```json
{
  "similar_works": [
    {
      "title": "已有论文标题",
      "source": "本地论文库 / arXiv",
      "similarity": "high | medium | low",
      "reason": "为什么认为相似"
    }
  ],
  "differentiation": "该 idea 与已有工作的主要差异",
  "risks": ["可能的撞题点"],
  "suggestions": ["下一步建议"]
}
```

### 5.4 UI 层

在 idea 卡片详情中新增「创新性检查」折叠卡片：

- 默认折叠，用户主动展开
- 展开后显示：
  - 相似工作列表（可点击跳转论文详情/外部链接）
  - 差异化建议
  - 风险提示
  - 「重新生成更独特的 idea」快捷操作

## 6. 实现路径

### 6.1 后端

1. **新增命令** `knowledge_check_idea_novelty`
   - 接收 idea 信息和检索范围（本地/外部）
   - 调用检索服务获取相似工作
   - 调用 LLM 生成 novelty 评估
   - 返回结构化结果

2. **复用服务**
   - `rag.rs` / `graph_rag.rs`：本地检索
   - 综述模块的 arXiv 检索：外部检索
   - `LlmClient`：novelty 评估

3. **新增 prompt**
   - `idea_novelty_prompt.rs` 或直接加入 `knowledge.rs`

### 6.2 前端

1. 在 `IdeaFromMaterialsPanel.tsx` / `TopicDiscoveryWizard.tsx` 的 idea 结果卡片中
   - 增加「检查创新性」按钮
   - 增加折叠面板展示 novelty 结果

2. 新增 hook（可选）
   - `useIdeaNoveltyCheck.ts`：管理加载、结果、错误状态

### 6.3 数据与配置

- 新增设置项：「idea 创新性检查默认使用外部检索」（默认关闭，避免 API 成本）
- 记录用户是否展开/使用 novelty 检查，用于后续 KR 统计

## 7. 验收标准

### 7.1 功能验收

- [ ] 对 `knowledge_ideas_from_materials` 生成的 idea，能点击「检查创新性」
- [ ] 本地检索能返回与 idea 相关的论文/笔记
- [ ] 外部检索（开启时）能返回 arXiv 相似论文
- [ ] LLM 评估结果包含相似工作、差异化建议、风险提示
- [ ] 结果展示不阻塞用户其他操作

### 7.2 非功能验收

- [ ] 本地检索平均耗时 < 2s（1000 篇论文以内）
- [ ] 外部检索失败时有明确降级提示
- [ ] 不泄露用户 idea 原文到外部（除非用户明确开启外部检索）
- [ ] type-check 与相关测试通过

## 8. 风险与应对

| 风险 | 影响 | 应对 |
| --- | --- | --- |
| LLM 误判相似性 | 中 | 只作为参考，不输出绝对结论；保留原始论文链接让用户自行判断 |
| 外部 API 成本 | 中 | 默认关闭外部检索；仅在用户主动开启时调用 |
| 本地论文库为空时检索无结果 | 低 | 提示「本地论文库为空，请先导入论文或开启外部检索」 |
| 增加用户认知负担 | 中 | 默认折叠，不强制展示；结果用语克制，避免制造焦虑 |

## 9. 后续可扩展

- 与 AgentRuntime 结合：把 novelty check 作为一个 Agent 节点，融入「选题规划」多 Agent 工作流
- 与投稿管理结合：在选定 idea 后自动生成「相关工作」写作素材
- 与知识图谱结合：把 idea 和相似论文建立证据链关系
