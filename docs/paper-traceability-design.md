# 文献逻辑溯源机制

## 定位

「文献逻辑溯源机制」是对现有论文精读与知识图谱能力的纵向延伸，帮助用户从"读一篇论文"升级为"追踪一条技术脉络"。

它支持两个互补维度：
- **横向对比**：多篇文献在方法、实验、数据集、指标上的差异（已有能力补强）
- **纵向溯源**：单篇重点文献的引用基石（ancestors）与衍生研究（descendants）的"顺藤摸瓜"

最终目标是构建可交互的技术演进时间线，实现科研情报的深度挖掘。

---

## 解决的核心痛点

| 痛点 | 说明 |
|---|---|
| 只见树木不见森林 | 读单篇论文时，不清楚该方法在整条技术线上的位置 |
| 引用追溯费时 | 手动翻查参考文献、查找施引文献效率低 |
| 演进关系模糊 | 难以判断 A 方法是在 B 基础上改了什么，C 又改进了 A 的哪一点 |
| 实验对比零散 | 跨论文的实验设置、数据集、指标口径不一致，难以横向比较 |

---

## 核心功能

### 1. 引用基石图谱（Backward Traceability）

针对单篇重点论文，自动解析其真正依赖的学术基石：

- **直接引用图**：从 PDF 参考文献列表 + 元数据中提取被引论文
- **关键引用识别**：通过 LLM 判断哪些被引文献是"方法来源"、哪些是"相关工作"、哪些是"通用背景"
- **引用上下文还原**：展示原文如何提及该参考文献（直接引用片段）
- **基石聚合**：向上追溯 2-3 层，形成"方法源头 → 关键改进 → 本文"的层级树

### 2. 衍生研究图谱（Forward Traceability）

自动追踪某篇论文发表后被引用/改进/对比的后续工作：

- **施引文献发现**：通过 Semantic Scholar / Crossref / OpenAlex 等开放 API 获取 citing papers
- **衍生关系分类**：LLM 判断每篇施引文献属于以下哪类
  - `直接改进`（improvement）
  - `应用拓展`（application）
  - `对比验证`（comparison）
  - `批评/质疑`（critique）
  - `引用背景`（background）
- **影响力衰减视图**：按年份展示引用量与重要衍生工作的分布

### 3. 技术演进时间线

将纵向溯源结果以时间轴形式呈现：

- 横轴：发表年份
- 节点：论文（大小可映射引用量或影响力）
- 边：方法继承 / 改进 / 对比关系
- 颜色/线型：表示关系类型（继承、改进、对比、批评）
- 交互：点击节点展开论文卡片，可一键导入论文库或发起精读

### 4. 多文献横向对比表

在溯源过程中，用户可选中多篇论文生成结构化对比：

| 对比维度 | 示例字段 |
|---|---|
| 方法 | 核心思想、模型结构、关键假设 |
| 实验 | 数据集、评测指标、 baseline |
| 结果 | 主指标数值、统计显著性 |
| 局限 | 作者自述局限、后续工作批评 |
| 代码 | 是否开源、框架、可复现性 |

## 技术实现要点

### 数据源

| 数据 | 来源 |
|---|---|
| 本地论文元数据 | SQLite `papers` 表 |
| 参考文献列表 | PDF 文本提取 + 正则/LLM 解析 |
| 引用关系 | Semantic Scholar API、OpenAlex API、Crossref（fallback） |
| 论文分类标签 | 本地 Graph RAG 知识图谱 + LLM 判断 |

### 关键算法/流程

1. **引用解析器**
   - 输入：PDF 全文或参考文献章节
   - 输出：结构化引用条目（标题、作者、年份、venue、DOI 候选）
   - 实现：lopdf 文本提取 + 参考文献专用正则 + LLM fallback

2. **引用关系匹配**
   - 将解析出的引用条目与本地库 + 开放 API 进行匹配
   - 生成统一 ID（本地 paper_id 或外部 S2Id / OpenAlexId）

3. **关系类型分类器**
   - 输入：目标论文 + 引用/被引论文的标题摘要
   - 输出：关系类型 + 一句话说明
   - 实现：LLM 少量示例提示（few-shot），可缓存结果

4. **时间线布局引擎**
   - 前端基于 D3 / ECharts / 自定义 Canvas 实现力导向或分层时间轴
   - 节点坐标按年份 + 影响力综合计算

### 数据模型建议

```sql
-- 引用关系表（有向边）
paper_citation_edges (
  source_id TEXT,           -- 引用方
  target_id TEXT,           -- 被引方
  relation_type TEXT,       -- inherits | improves | applies | critiques | background
  evidence_quote TEXT,      -- 原文引用片段（可选）
  confidence REAL,          -- 匹配/分类置信度
  source_origin TEXT,       -- local_pdf | semantic_scholar | openalex | user
  created_at INTEGER,
  PRIMARY KEY (source_id, target_id)
);

-- 论文溯源快照（按重点论文聚合）
paper_traceability_snapshots (
  paper_id TEXT PRIMARY KEY,
  ancestor_tree JSON,       -- 向上追溯树
  descendant_tree JSON,     -- 向下衍生树
  timeline_data JSON,       -- 时间线渲染数据
  last_refreshed INTEGER
);
```

## 与现有模块的关系

- **论文精读**：溯源入口可嵌入论文详情页，精读结果自动作为节点摘要
- **知识图谱 / Graph RAG**：引用关系边可直接写入现有图谱，增强多跳推理
- **研究领域动态**：时间线视图可与"领域演进"功能共享前端组件与后端聚合逻辑
- **arXiv 智能检索**：施引文献中发现的重要新论文可一键导入论文库
- **投稿管理**：回复审稿人时可用溯源结果快速定位"相关工作"

## 实现阶段

### MVP（4-6 周）

- [ ] 实现 PDF 参考文献解析器（支持常见格式）
- [ ] 接入 Semantic Scholar API 获取被引/施引关系
- [ ] 在论文详情页新增「引用关系」标签页
- [ ] 展示基础引用列表与层级树（ ancestors 1-2 层 ）
- [ ] 支持将外部论文一键导入本地库

### 进阶（6-10 周）

- [ ] 接入 OpenAlex / Crossref 作为 fallback 与交叉验证
- [ ] LLM 关系分类器（方法继承、改进、应用、批评）
- [ ] 可交互技术演进时间线
- [ ] 多文献横向对比表生成
- [ ] 支持用户手动校正/补充引用关系

### 未来扩展

- [ ] 与 LLM 对话结合："这篇论文的方法源头是什么？"
- [ ] 自动发现"被忽视但重要"的基础工作
- [ ] 跨语言文献溯源（中文论文追踪英文源头）

---

## 验收标准

- [ ] 对本地重点论文，能在 3 秒内展示其直接引用与施引列表
- [ ] 关系分类准确率（人工抽样 50 条）达到 80% 以上
- [ ] 时间线支持 50 个节点流畅交互
- [ ] 新增数据表与现有 Graph RAG  schema 无冲突
