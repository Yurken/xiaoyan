# 小妍 — 项目概述与核心亮点

## 一、项目概述

**小妍**是一款面向研究生与科研工作者的桌面端 AI 研究助手，采用 "桌面端本地内核" 架构（Tauri v2 + Rust + SQLite），以 Supervisor 驱动的多 Agent 编排为核心，覆盖研究方向规划、文献调研、论文精读、复现建议、知识库沉淀和带引用的对话问答等科研全链路。

**产品定位**：不是单一的聊天工具，而是一套围绕科研主流程设计的"研究操作系统"。

**三端定位**：
- **桌面端（旗舰端）**：全功能，本地自包含，隐私可控
- **Web 端**：展示版 / 远程协作版
- **移动端**：轻量陪伴端，消费型场景

## 二、核心亮点

### 亮点 1：科研全链路一体化工作台

市面现有工具多为"单点能力"（ChatPDF 读论文、Zotero 管文献、ChatGPT 问问题），用户需要在多个工具间反复切换，上下文不断丢失。小妍将科研全链路整合为统一工作台：

```
选题规划 → 文献调研 → 论文精读 → 知识沉淀 → 实验记录 → 投稿管理
```

每个环节的数据可自动流转到下一环节，形成"连续推进、全程留痕、可回溯复用"的科研闭环。

### 亮点 2：可观测的多 Agent 协同架构

小妍不是"一个模型回答所有问题"，而是采用 **Supervisor 驱动的多 Agent 编排**：

- **Supervisor Agent**：负责任务拆解与路由决策
- **检索 Agent**：文献检索与证据收集
- **规划 Agent**：研究方向拆解与学习路线生成
- **文献侦察 Agent**：论文筛选与摘要分析
- **综述 Agent**：结构化综述写作
- **论文解析 Agent**：PDF 精读与图表关联分析
- **复现 Agent**：实验复现指导

用户在小妍协同台不仅得到最终答案，还能实时查看**计划步骤、Agent 执行轨迹、中间产物与来源引用**，大幅提升 AI 协作的可解释性与可信度。

### 亮点 3：本地化隐私优先架构

与现有云端科研工具不同，小妍采用**完全自包含的本地架构**：

- **前端**：React + TypeScript + Vite
- **桌面容器**：Tauri v2
- **后端能力层**：Rust Command（LLM 调用、RAG 检索、Agent 编排）
- **数据层**：本地 SQLite

所有论文、笔记、会话、实验与投稿数据存储在本地，敏感 API Key 采用掩码策略，设置支持 AES-256-GCM 加密导出。用户无需担心研究数据泄露。

### 亮点 4：知识图谱 + Graph RAG 增强检索

小妍不仅做向量相似度检索（RAG），还构建了**论文引用关系知识图谱**：

- **citation_graph.rs**：自动构建论文引用关系图（节点 + 有向边）
- **graph_rag.rs**：检索时融合图邻域上下文

这意味着当用户提问"与某篇论文方法相似的工作有哪些"时，系统不仅靠语义相似度，还能通过引用关系图进行多跳推理，回答质量显著优于纯向量检索。

### 亮点 5：视觉模型驱动的 PDF 图表深度解析

论文精读不仅是文本分析。小妍在 PDF 图表解析上做了双重保障：

- **lopdf 提取**：提取 PDF 中的位图图像
- **视觉模型扫描**（"视界"模块）：调用 GPT-4o 等视觉模型扫描 PDF 各页，识别 lopdf 无法提取的矢量图与表格

AI 分析时积极引用图表编号，结果以"文字 + 图片并排"呈现，让论文精读真正"读图"而不只是"读文字"。

### 亮点 6：AI 模拟审稿与投稿全生命周期管理

小妍将"投稿管理"从简单的待办事项升级为完整的项目管理：

- **DDL 日历**：追踪会议与期刊截止日期
- **投稿看板**：Kanban 视图管理投稿状态
- **提交清单**：投稿前检查项
- **版本管理**：PDF 上传/下载、版本快照
- **AI 模拟审稿**：基于 PDF 全文提取生成 2-4 位模拟审稿人意见（含大修/小修/接收/拒稿结论）
- **审稿归档**：审稿意见跟踪与回复策略沉淀

这是国内科研助手工具中较为完整的投稿工作流支持。

### 亮点 7：细粒度的模型分工配置

小妍支持按用途自定义大模型分工，而非"一个模型包打天下"：

| 分组 | 用途 |
|---|---|
| 快速模型 | 方向提示、多 Agent 调度、轻量对话 |
| 深度分析模型 | 规划分析、论文精读、复杂研究分析 |
| 写作整合模型 | 综述写作、最终整合回答 |
| 代码复现模型 | 复现指导、训练配置 |
| 视觉模型 | PDF 图表扫描识别 |

用户可按成本、延迟、能力偏好灵活搭配不同服务商（OpenAI / Anthropic / 阿里云 DashScope / DeepSeek 等）。

## 三、解决的科研痛点

| 痛点 | 小妍的解决方案 |
|---|---|
| 研究流程割裂 | 一体化工作台，10 个模块统一界面 |
| 研究资产难沉淀 | 论文、笔记、会话、实验、投稿统一落库存储 |
| AI 使用不可解释 | 多 Agent 执行轨迹可视化，计划步骤可回溯 |
| 数据隐私担忧 | 本地 SQLite 存储，无需上传论文到云端 |
| 工具切换成本高 | 阅读 → 笔记 → 对话 → 投稿在同一应用内完成 |
