# 小妍答辩 PPT 必须体现的功能清单

> 以下功能是答辩评委最可能关注、最能体现项目技术深度与科研场景价值的点，PPT 中必须至少用一页或一个 demo 场景展示。

---

## 一、科研全链路闭环（必须讲清楚定位）

### 1. 工作台首页
- 状态驱动的下一步行动推荐；
- 根据研究主题阶段、论文分析进度、知识沉淀、投稿截止等生成今日 agenda；
- 体现"回到首页就知道下一步"。

### 2. 选题规划
- 输入研究兴趣和关键词；
- 输出结构化研究路线、阶段目标、核心问题；
- 与后续论文库、知识库、对话打通。

### 3. 文献调研
- arXiv 三步级联检索（领域 → 类型 → 等级）；
- 内置 CCF 目录、期刊分区库辅助筛选；
- 搜索结果可一键导入论文库。

### 4. 论文精读
- 上传 PDF 后自动提取全文、图表；
- 输出多维度分析：研究问题、方法、实验、局限、复现提示；
- 结果沉淀为可复用资产。

### 5. 知识沉淀
- 论文分析结论一键转为知识笔记；
- 知识图谱视图：观点、证据、关系；
- 支持跨论文证据链构建。

### 6. 实验记录
- 结构化字段：目标、配置、结果、附件；
- 可与论文、投稿关联，形成证据链。

### 7. 投稿管理
- DDL 日历 + Kanban 看板；
- 版本快照、审稿意见归档；
- AI 模拟审稿（2-4 位审稿人多维度评审）。

---

## 二、三大核心技术创新（必须单独成页）

### 8. 多智能体协同编排
- Supervisor（任务调度中枢）+ 7 个专业 Agent（智能体）；
- 任务拆解、依赖图调度、波次并行；
- 执行轨迹可观测（Mission Control 面板）。

### 9. Graph RAG（图增强检索生成）
- 向量语义检索 + 论文引用关系图；
- 支持多跳推理，如"某方法被哪些后续工作改进"；
- 区别于普通 ChatPDF/ChatGPT 的关键能力。

### 10. 双路 PDF 图表解析
- lopdf（PDF 解析库）提取位图和标题；
- 视觉模型扫描矢量图和复杂表格；
- 论文精读时图文并排、积极引用图表编号。

---

## 三、隐私与本地架构（差异化卖点）

### 11. 本地内核
- Tauri（跨平台桌面应用框架）+ Rust（系统级编程语言）+ SQLite（本地轻量级数据库）；
- 完全离线可用，论文数据不上云。

### 12. 隐私保护
- API Key（调用外部模型服务的密钥）前端掩码、后端持有；
- 应用锁、记忆隐私分层；
- 配置加密导出（AES-256-GCM + PBKDF2）。

---

## 四、交互与体验亮点（加分项）

### 13. 小妍对话（跨域协同入口）
- 基于本地论文库、知识图谱、长期记忆回答；
- 答案带来源引用；
- 复杂问题触发多 Agent 协同。

### 14. 长期记忆
- 手动备忘 + 近期操作 + AI 观察三层；
- 对话时自动注入相关记忆；
- 让 AI"记得"用户研究背景。

### 15. 桌面伴侣
- Canvas 动画状态反馈；
- Agent 执行中/完成/出错等状态可视化；
- 科研陪伴感，区别于工具类产品。
