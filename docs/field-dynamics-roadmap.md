# 研究领域动态功能规划

## 定位
「研究领域动态」是「研究主题规划」的组成部分，帮助用户持续跟踪已关注研究领域的最新进展，以**周期性简报**为当前核心形态，未来逐步扩展为更完整的领域情报中心。

## 当前已实现（MVP）
- **领域动态简报**：为每个 `research_interest` 自动生成一份简报，包含：
  - 本期综述（2-4 句话）
  - 热点/趋势标签（3-6 条）
  - 关键论文（3-6 篇，支持导入到论文库）
  - 相关会议截稿提醒（未来 90 天内）
- **数据来源**：arXiv、Semantic Scholar、本地 venues（CCFDDL）
- **触发方式**：启动后自动扫描 + 手动刷新
- **入口**：整合在「规划」页面中

## 后续规划

### 阶段一：领域健康度看板
把单份简报扩展为可感知领域「活跃度」的看板。

- 每个研究兴趣展示：
  - 近期论文数量趋势（近 4 周/12 周折线）
  - 高频关键词云 / 新兴关键词提示
  - 核心作者 / 活跃机构榜单
  - 热门会议/期刊分布
- 技术要点：
  - 在 `research_field_briefings` 中增加统计快照字段，或新增 `research_field_stats` 表
  - 复用扫描时收集的论文元数据进行聚合
  - 前端使用轻量图表组件（建议优先用现有 UI 库或纯 CSS/SVG，避免引入重型图表库）

### 阶段二：跨领域趋势对比
支持同时选择多个研究兴趣，发现交叉趋势。

- 功能：
  - 多选兴趣进行对比，展示各领域论文量、热度、关键词重叠度
  - 自动识别「交叉热点」：多个领域共同出现的新方法或关键词
  - 推荐潜在的研究交叉方向
- 技术要点：
  - 后端增加跨领域聚合接口
  - LLM 根据多领域简报生成交叉分析
  - 前端增加对比选择器与对比视图

### 阶段三：时间线视图
以时间轴形式呈现领域演进，帮助用户感知长期趋势。

- 功能：
  - 按周/月聚合每个领域的论文、会议、热点事件
  - 展示「重要节点」：高引用论文、突破性方法、重要会议召开
  - 支持回溯查看历史简报
- 技术要点：
  - `research_field_briefings` 已保留 `period_start`/`period_end`，可扩展为历史表
  - 将 upsert 改为 append，定期清理过期数据
  - 前端使用时间轴组件或自定义垂直时间线

## 数据表演进建议
```sql
-- 当前
research_field_briefings (
  interest_id UNIQUE,  -- 每个兴趣仅保留最新简报
  ...
)

-- 阶段三可演进为
research_field_briefings (
  interest_id,
  period_start,
  period_end,
  ...
  PRIMARY KEY (interest_id, period_start)
)
```

## 与相关功能的关系
- **Active Researcher（arXiv 扫描）**：侧重「发现新论文并导入」，是论文工作流的入口。
- **研究领域动态**：侧重「理解领域态势」，是规划/决策的辅助。
- **Knowledge（研究兴趣）**：提供关注领域的权威来源，动态简报基于这些兴趣生成。
