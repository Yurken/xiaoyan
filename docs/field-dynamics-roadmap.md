# 研究领域动态功能规划

## 定位
「研究领域动态」是「研究主题规划」的组成部分，帮助用户持续跟踪已关注研究领域的最新进展，以**周期性简报**为当前核心形态，未来逐步扩展为更完整的领域情报中心。

## 当前已实现（MVP）
- **领域动态简报**：为每个 `research_interest` 自动生成一份简报，包含：
  - 本期综述（2-4 句话）
  - 热点/趋势标签（3-6 条）
  - 关键论文（3-6 篇，支持导入到论文库）
  - 相关会议截稿提醒（未来 90 天内）
- **数据来源**：arXiv、Semantic Scholar、本地 venues（CCFDDL）
- **触发方式**：启动后自动扫描 + 手动刷新
- **入口**：整合在「规划」页面中

## 本轮升级基线（已完成）

本轮先解决“简报能生成”之外的连续使用问题，再继续扩展更重的情报分析能力：

- **可回溯**：每次成功生成都会追加不可变快照；当前简报仍保持每个兴趣一份，历史简报可单独查看和导入论文。
- **可感知**：简报保存候选论文、精选论文、截稿和热点数量，前端据此汇总当前领域活跃度，并展示最近采样的论文量变化。
- **结果可信**：候选论文跨来源去重；LLM 只能从实际候选中选择论文和会议，标题、链接和导入标识始终以原始数据为准；模型不可用或返回非结构化结果时，使用可读的确定性简报降级。
- **刷新稳定**：扫描接口返回完整的当前简报集，不会因某一兴趣本次无结果而清空旧卡片；刷新后使用已持久化的 ID，避免论文导入指向不存在的简报。

### 后续优先级

1. **高价值信号（下一步）**：持久化原始候选元数据，补充按周的关键词、新兴词、作者、机构和 venue 分布；当前“候选论文量”仅反映检索量，不能当作引用影响力。
2. **跨领域对比**：在至少积累 4 个有效周快照后，提供多选领域的数量归一化、关键词重叠和交叉机会，避免用稀疏数据得出趋势结论。
3. **可操作闭环**：为论文和截稿加入收藏/忽略、研究任务和通知规则，让简报结果进入用户的阅读与规划工作流。
4. **数据治理**：为历史快照加入保留策略、导出和明确的数据来源/扫描失败状态，保证长期使用时可解释、可控。

## 后续规划

### 阶段一：领域健康度看板
把单份简报扩展为可感知领域「活跃度」的看板。

- 每个研究兴趣展示：
  - 近期论文数量趋势（近 4 周/12 周折线）
  - 高频关键词云 / 新兴关键词提示
  - 核心作者 / 活跃机构榜单
  - 热门会议/期刊分布
- 技术要点：
  - 在 `research_field_briefings` 中增加统计快照字段，或新增 `research_field_stats` 表
  - 复用扫描时收集的论文元数据进行聚合
  - 前端使用轻量图表组件（建议优先用现有 UI 库或纯 CSS/SVG，避免引入重型图表库）

### 阶段二：跨领域趋势对比
支持同时选择多个研究兴趣，发现交叉趋势。

- 功能：
  - 多选兴趣进行对比，展示各领域论文量、热度、关键词重叠度
  - 自动识别「交叉热点」：多个领域共同出现的新方法或关键词
  - 推荐潜在的研究交叉方向
- 技术要点：
  - 后端增加跨领域聚合接口
  - LLM 根据多领域简报生成交叉分析
  - 前端增加对比选择器与对比视图

### 阶段三：时间线视图
以时间轴形式呈现领域演进，帮助用户感知长期趋势。

- 功能：
  - 按周/月聚合每个领域的论文、会议、热点事件
  - 展示「重要节点」：高引用论文、突破性方法、重要会议召开
  - 支持回溯查看历史简报
- 技术要点：
  - `research_field_briefings` 已保留 `period_start`/`period_end`，可扩展为历史表
  - 将 upsert 改为 append，定期清理过期数据
  - 前端使用时间轴组件或自定义垂直时间线

## 数据表演进建议
```sql
-- 当前
research_field_briefings (
  interest_id UNIQUE,  -- 每个兴趣仅保留最新简报
  ...
)

-- 阶段三可演进为
research_field_briefings (
  interest_id,
  period_start,
  period_end,
  ...
  PRIMARY KEY (interest_id, period_start)
)
```

## 与相关功能的关系
- **Active Researcher（arXiv 扫描）**：侧重「发现新论文并导入」，是论文工作流的入口。
- **研究领域动态**：侧重「理解领域态势」，是规划/决策的辅助。
- **Knowledge（研究兴趣）**：提供关注领域的权威来源，动态简报基于这些兴趣生成。
