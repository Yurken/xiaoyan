### 第 2 页：研究背景与痛点

**现有科研工具的三大痛点**：
1. **流程割裂**：规划、检索、阅读、写作、投稿分散在多个工具
2. **资产分散**：论文、笔记、对话、实验结论难以统一管理
3. **AI 黑盒**：仅有结果，没有过程与来源

**配图建议**：工具切换示意图，展示用户在不同应用间跳转的混乱状态

---

### 第 3 页：项目定位

**小妍是什么？**

> 不是单一的聊天工具，而是一套围绕科研主流程设计的"研究操作系统"

**覆盖链路**：
```
选题规划 → 文献调研 → 论文精读 → 知识沉淀 → 实验记录 → 投稿管理
```

**核心价值主张**：
- 连续推进（不丢上下文）
- 全程留痕（可追溯）
- 资产沉淀（可复用）

### 第 6 页：创新点一 — 多智能体协同编排

**问题**：单一模型包打天下，面临能力边界、上下文限制、可解释性三重瓶颈

**方案**：
- 中央调度器负责任务拆解与流程编排
- 7+ 个专业智能体（检索 / 规划 / 解析 / 综述 / 复现 …）
- 三种路由模式（规则 / 模型 / 混合）

**展示点**：任务控制中心面板截图，展示智能体执行轨迹

---

### 第 7 页：创新点二 — 知识图谱增强检索

**问题**：传统向量检索无法回答多跳引用关系问题

**方案**：
- 引用关系图引擎：构建论文引用网络
- 图增强检索：检索时融合图邻域上下文，支持多跳推理

**示例问题**：
> "哪些论文引用了 Transformer 并改进了其注意力机制？"

纯向量检索 → 语义相似但无引用关系
图谱增强检索 → 沿引用图多跳推理，返回真正相关的后续工作

---

### 第 8 页：创新点三 — 视觉模型图表解析

**问题**：传统方案只能提取位图，矢量图和表格容易丢失

**方案**：双路并行

| 路 A：规则提取 | 路 B：视觉模型扫描 |
|---|---|
| 提取位图图像 | 识别矢量图、复杂表格 |
| 匹配图表标题 | 输出编号、类型、描述 |

**展示点**：论文精读界面截图，"如图 3 所示" + 图文并排

### 第 13 页：数据与隐私安全

**本地优先原则**：
- 论文、笔记、会话、实验、投稿 → 本地加密存储
- API Key → 掩码展示，后端持有真实值
- 配置导出 → 行业级加密算法保护

**多层隐私保护**：
- **应用锁**：闲置超时自动锁定，密码门控
- **记忆隐私**：敏感记忆可独立设置查看密码，分层访问控制
- **本地离线**：无网络也能查阅，不上传论文到云端

**三端数据路径**：
```
Desktop：本地存储（完全离线可用，多层隐私保护）
Web：远程后端（协作场景）
Mobile：远程后端（轻量查阅）
```
