# 更新日志

## [0.3.2] - 2026-05-05

### 新增
- 投稿时间线与拒稿转投恢复面板：投稿管理新增时间线视图，支持拒稿后一键转投恢复与进度回溯。
- 设置历史与配置切换：支持保存/恢复多组配置快照，快速切换不同工作场景（如论文精读 / 本地 Ollama / Survey 写作）。
- 小妍宣传片 Remotion 工程：新增基于 Remotion 的视频宣传片工程，用于产品演示与推广。
- Kimi For Coding 与搜索引擎配置：支持 Kimi 编码助手接入，独立配置搜索引擎参数与能力域模型。
- 墩墩状态联动：XiaoYanPet 宠物状态与系统运行状态联动，强化研究流程中的状态感知。
- macOS 窗口拖动条：顶部新增原生窗口拖拽区域，适配 macOS 标题栏交互规范。
- 前端设计插件体系：引入 `frontend-design` 插件，支持设计系统规范的一键应用与组件规范化。

### 优化
- Copilot 界面与交互优化：优化对话界面布局、流式输出展示与多会话管理交互，提升长对话可读性。
- 投稿人侧智能匹配规划：融入投稿人侧功能规划与智能匹配路线，完善投稿决策链路。
- 品牌与图标全面更新：ResearchCopilot → 小妍（xiaoyan）品牌统一，全端图标、Logo、文案与视觉资产刷新。
- 工作台交互优化：工作台布局密度、间距节奏与头部结构优化，统一视觉层级。
- 多页面文案提升：优化首页、设置、工具等页面的文本描述，降低理解成本。

### 修复
- 桌面端状态恢复与错误反馈：补强页面状态恢复机制，收敛 `alert()`/`console.error` 混用，提升异常场景可恢复性。
- 布局切换后路由失效：修复 Landscape/Focus 双布局切换后的路由映射问题，避免特定页面落入 `*` 重定向。
- 设置默认值与历史操作竞态：修复设置保存与配置历史操作的并发冲突，降低配置丢失风险。
- Copilot 流式请求生命周期：收敛流式请求状态管理，修复 Abort 信号处理与异常中断问题。
- Copilot.tsx 编码与标签修复：修复合并后的 UTF-8 编码损坏与未闭合 `p` 标签导致的渲染异常。
- Windows 图标与签名：提升 Windows 任务栏图标清晰度，修复更新包签名与圆角图标显示问题。
- macOS 拖拽与配色：修复聚焦后顶部拖拽失效，优化窗口拖动条交互配色与层级。
- 禁止桌面端图标拖拽：防止侧边栏 Logo、头像等元素被意外拖出窗口。

### 重构
- 拆分 LLM 传输逻辑：抽离 LLM 传输层为独立模块，增强设置错误处理与连接测试稳定性。
- 品牌资产重构：更新应用图标、启动图、侧边栏链接与 shell 布局，收敛品牌视觉一致性。

### 工程化
- 代码审计报告：新增桌面端代码审计与页面流转可用性审计文档，覆盖路由、数据流、交互一致性与可访问性。
- Kimi 协作约束：补充 Kimi 编码协作规范与约束文档，明确大文件治理与 feature 分层标准。

## [0.3.1] - 2026-04-19

### 新增

- 实验记录删除确认：实验页删除操作新增危险确认弹窗与删除中状态，避免误删。

### 优化
- 论文解析链路优化：收敛历史解析实现并补强预览文本与全文提取回退策略，提升异常场景可恢复性。
- 论文生成流程增强：支持双源生成并优化修正文稿流程，降低中间状态中断概率。
- 知识协同体验优化：打通知识图谱与笔记联动，改进研究路径与笔记沉淀衔接效率。
- 桌面端界面一致性提升：统一主要页面头部比例、间距节奏与背景层级，收敛阴影与背景色主题 token。

- 首页拟态一致性补齐：补齐“研究进展”“最近研究方向”“最近知识卡片”“最近对话”子卡片拟态层次。
- 主题行为收敛：拟态增强仅在非极简风格生效，`modern-minimal` 极简主题保持原有视觉表现。

### 修复
- 论文解读稳定性：修复空结果、关键词清洗与 Markdown 落盘问题，并修复综述检索与解析状态错误。
- 论文导入容错：预览 PDF 解析失败时自动回退全文，降低导入后无法继续分析的概率。
- 投稿与审稿流程：修复投稿版本计数串台与 AI 审稿结果 Markdown 渲染异常。
- 知识与协同交互：修复知识图谱智能布局问题，改进任务总览状态图交互与宠物视线跟踪表现。
- 设置与可访问性：修复 Anthropic 兼容端点测试、表单可访问性与确认交互细节。
- 主题与样式细节：修复未选中 Toggle 背景色、主题默认值与侧边栏层级问题。
- 工程配置与请求标识：补充 `typeRoots/types` 类型配置，网页剪藏 UA 标识更新为 `XiaoYan/1.0`。

### 工程化

## [0.3.0] - 2026-04-12

### 新增
- 投稿管理与实验工作流：新增独立的投稿页与实验页，覆盖会议/期刊追踪、投稿看板、提交清单、版本管理、审稿归档与实验记录。
- 投稿版本能力：支持论文版本 PDF 上传/下载、版本快照保存，以及在版本维度触发 AI 模拟审稿。
- AI 模拟审稿生成器：基于 PDF 文本提取生成结构化审稿意见，支持结果落库并进入审稿回复流程。
- 知识图谱与 Graph RAG：新增引用关系图构建、图谱检索增强（Graph RAG）与知识图谱工作区。
- 小妍协同台可观测能力：新增 Agent 状态图与运行轨迹展示，提升多 Agent 协作过程可解释性。
- 论文处理链增强：新增 PDF 文本提取、图表提取、图表引用解析、关键词提取与关键词回填等能力。
- 论文导入自动重命名：新增设置项，支持按策略自动重命名导入 PDF 文件。
- 记忆管理：新增手动/自动记忆录入与设置面板，支持对话上下文增强。
- 设置安全迁移：新增配置导入/导出能力，支持加密配置文件迁移。
- 工具能力扩展：新增学术翻译、Markdown 格式化、来源查询、期刊分区查询等工具能力。
- PPT 工作流增强：新增 PPT 生成与幻灯片预览面板，支持在工作区直接预览导出前内容。
- 小妍桌面伴侣：新增多状态动画组件与场景化反馈，强化研究流程中的状态感知。

### 重构
- 页面职责下沉：将 `Submission`、`Tools`、`Settings` 等大页面拆分到 `features/*`，推进“页面组合 + feature 逻辑”的结构化实现。
- 设置域重构：拆分连接配置、角色分工、技能、记忆、加密配置等子模块，降低单文件复杂度。
- 投稿域重构：将场所追踪、看板、清单、版本、审稿等能力拆为独立 workspace/panel/modal 组件。
- 知识域重构：重构知识图谱工作区，新增下拉切换与多面板协作结构（图谱、声明、引用、时间线、检查器）。
- UI 基础库统一：`packages/ui` 统一 Button/Input/Card/Select 等拟态组件，Web 端迁移共享组件以减少重复实现。
- Agent 执行图重构：小妍工作流改为显式状态机实现，调度链路更可控。

### 优化
- 品牌与文案统一：产品名称从“智研助手”统一为“小妍”，并同步导航、设置、对话与提示文案。
- 交互体验改进：优化首页、小妍、论文、实验、设置等页面布局密度与间距节奏，统一头部结构与视觉层级。
- 主题与外观能力：新增布局风格选择、现代极简风格支持，并持续收敛主题变量与阴影 token。
- 工具体验提升：arXiv 检索增强（结构化请求、防抖、检索范围联动），友链与来源查询交互优化。
- 论文处理稳定性：优化上传/解析流程与状态展示，减少大文件和异常文本导致的中断。
- 能力配置扩展：新增视觉模型、翻译模型、Semantic Scholar Key 等配置项，增强检索与解读的上限。

### 修复
- 修复 `sync-version` 脚本错误更新 `package-lock.json` 的问题（pnpm 项目无该文件，避免 CI `ENOENT`）。
- 修复发布链路中的 Windows 产物收集与 workflow artifacts 发布问题，提升多平台发布成功率。
- 修复设置保存事务与若干界面交互细节问题，降低配置丢失和状态不一致风险。

### 工程化
- 新增开发约束文档与重构路线文档，明确“大文件治理 + feature 分层”执行标准。
- 完善桌面端自动更新与发布流程（GitHub Release / 更新服务器），并补充版本同步脚本能力。
- 引入并统一多端共享类型与组件的使用边界，改善 Desktop/Web/Mobile 的协作成本。
